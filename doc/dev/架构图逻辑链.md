# 云枢观策平台技术架构图

## 一、客户端（MVVM 架构）

包含：

* 展示首页
* 政策广场
* 智能体对话
* 数据大屏
* 可视化图谱
* 个人中心
* 管理后台
* …

### 2\. 状态层（ViewModel）

包含：

* Pinia 状态管理
* 用户信息
* 鉴权令牌
* 主题/色系
* 会话状态
* 系统设置
* 权限路由
* …

### 3\. 接口层（Model / API）

包含：

* Axios HTTP 客户端
* WebSocket 长连接
* RESTful API 调用
* 流式数据接收
* 文件上传/下载
* 实时消息推送

### 客户端内部逻辑链

```text
表现层 View
    ↓↑
状态层 ViewModel
    ↓↑
接口层 Model / API
```

客户端通过：

```text
HTTPS / WebSocket
```

与服务端通信。

# 二、服务端（FastAPI）

服务端整体分成两条核心链路：

```text
接口-业务-数据链路
+
接口-智能体-记忆-工具链路
```

\---

## 1\. 接口-业务-数据链路

### 1.1 接口层（API Gateway）

包含：

* 路由管理
* 鉴权认证 JWT / OAuth2
* 限流控制
* 参数校验 Pydantic

### 1.2 业务层（Business Layer）

包含：

* 用户与权限
* 政策解析
* 材料推荐
* 流程指引
* 统计分析
* 通知与邮件
* 新闻资讯
* …

### 1.3 数据访问层（ORM / DAO）

包含：

* SQLModel / SQLAlchemy
* 事务管理
* 数据映射
* CRUD 封装

### 业务数据链路逻辑

```text
客户端请求
    ↓
API Gateway 接口层
    ↓
Business Layer 业务层
    ↓
ORM / DAO 数据访问层
    ↓
关系型数据库 RDBMS
```

这条链路主要处理传统业务逻辑，例如用户权限、政策解析、材料推荐、流程指引、统计分析、通知邮件等。

\---

## 2\. 接口-智能体-记忆-工具链路

### 2.1 接口层（Agent API）

包含：

* 对话接口
* 上传解析
* 图谱生成
* 流式响应

### 2.2 智能体层（Agent / LangGraph）

包含：

* 任务规划
* 思考推理（CoT）
* RAG 检索增强
* 工具调用
* 结果生成
* 流式输出

### 2.3 记忆层（Memory）

包含：

* 短期会话记忆
* 长期向量记忆
* 用户画像记忆

### 2.4 工具层（Tools）

包含：

* 政策检索工具
* 文件解析工具
* 图谱构建工具
* 外部 API 工具
* …

### 智能体链路逻辑

```text
客户端对话 / 文件上传 / 图谱请求
    ↓
Agent API
    ↓
Agent / LangGraph 智能体层
    ↓
任务规划 / 推理 / RAG / 工具调用
    ↓
记忆层 Memory + 工具层 Tools
    ↓
外部服务 / 向量数据库 / 缓存服务
    ↓
结果生成 / 流式输出
    ↓
返回客户端
```

这条链路主要负责 AI 对话、政策问答、RAG 检索、文件解析、图谱生成、智能推荐、流式响应等能力。

# 六、完整逻辑链总览

可以把整张图理解为下面这一条完整主链：

```text
用户在客户端输入请求
    ↓
客户端 View 接收操作
    ↓
ViewModel 管理状态、用户信息、鉴权令牌、会话状态
    ↓
Model / API 通过 Axios 或 WebSocket 发起请求
    ↓
服务端 FastAPI 接收请求
    ↓
API Gateway 进行路由、鉴权、限流、参数校验
    ↓
根据请求类型分流
```

## 分流一：普通业务请求

```text
API Gateway
    ↓
Business Layer
    ↓
ORM / DAO
    ↓
RDBMS / Cache
    ↓
返回业务结果
    ↓
客户端展示
```

典型场景：

```text
用户登录
权限校验
政策列表查询
材料推荐
流程指引
统计分析
后台管理
通知邮件
```

## 分流二：智能体请求

```text
Agent API
    ↓
Agent / LangGraph
    ↓
任务规划
    ↓
RAG 检索增强
    ↓
调用向量数据库 / 记忆层 / 工具层 / 外部模型
    ↓
结果生成
    ↓
流式输出
    ↓
客户端实时展示
```

典型场景：

```text
智能体对话
政策问答
文件上传解析
政策图谱生成
用户画像分析
材料智能推荐
```

# 七、架构核心关系总结

这张图的核心思想是：

```text
前端采用 MVVM
后端采用 FastAPI 分层架构
业务链路处理传统系统功能
智能体链路处理 AI / RAG / 工具调用能力
数据层同时支持关系型数据、向量数据和缓存数据
外部服务提供大模型、Embedding、OCR、邮件和第三方 API 能力
```

最终形成：

```text
客户端交互
    ↓
服务端鉴权与路由
    ↓
业务处理 + 智能体推理
    ↓
数据库 / 向量库 / 缓存 / 外部服务协同
    ↓
结构化结果或流式结果返回客户端
```
