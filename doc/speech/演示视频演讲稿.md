# 大纲  
### PPT  
PPT需要配备首页+痛点+宗旨+业务主要流程+部署网址,然后直接进入项目演示环节.  
##### 痛点

- **入口分散**：政策文件、基层通知和办事指南分布在多类渠道，用户获取成本高、触达效率低。
- **理解困难**：政策文本篇幅较长、表述专业，适用对象、办理条件、材料清单等关键要素不够突出。
- **办理断链**：政策阅读与实际申报、材料准备、进度管理之间缺少衔接，执行成本较高。
- **反馈缺失**：政策发布后的阅读情况、办理效果和用户反馈难以沉淀，服务优化依据不足。

##### 项目宗旨

- **统一共享入口**：构建云端政策共享平台，集中汇聚政策文件、基层通知、办事指南和认证主体发布内容，形成统一、稳定、可访问的政策信息入口。
- **智能政策解读**：融合 OCR、知识图谱、RAG 检索增强生成和智能体技术，将复杂政策转化为蕴含最优结构内容的自由知识图谱。
- **办事流程辅助**：打通政策理解、材料准备、待办生成、进度管理和结果沉淀链路，帮助企业与群众从“看懂政策”进一步走向“高效办理事项”。
- **反馈协同治理**：沉淀用户评论、解析纠错、办理反馈和行为数据，为政策发布主体与管理者提供效果追踪、问题识别和公共服务持续优化依据。

> 云枢观策致力于以人工智能赋能政策信息流转链路，构建“共享、解读、服务、协同”一体化平台，让企业与基层政策更加可达、可懂、可办、可行。
##### 业务流程图
从上到下:
- 企业政策发布
- 基层服务落地
- 政策可视化
- 待办与执行
侧边接入"云小圆"智能体.

10. 办事进度: 代办系统,可以新建待办,点击并完成待办,删除待办,还可以根据自己喜好切换是否已完成按钮.将用户对于政策从了解和学习无缝衔接到实践的环节.  
11. 数据分析页面:简要概括.  
12. 云小圆: 重中之重.  
    1. 先介绍实现智能体的流程: 将LLM(Kimi)作为智能体的核心决策驱动,然后配备决策内循环+长短记忆+工具调用+意图控制展示+强审核机制:  
    2. 决策内循环: 采用ReAct架构,基于LangChain和LangGraph制作了模型驱动+模型思考+模型观察的自我循环,使得模型从一个输出者跃迁为一个决策者.  
    3. 长短记忆: 智能体的短期记忆采用"上下文裁剪+关键信息提取"的混合模式,长期记忆则采用了RAG检索增强生成+ChromaDB向量库的模式.  
          1. 在这个过程中,关键对话信息和历史记录将通过Embedding模型转化为向量化的坐标.当智能体调用搜索工具时,用户的问题也会被转化为同一向量空间中的查询向量,再在 ChromaDB 向量库中基于向量相似度检索 Top-K 语义邻近片段,从而实现意图搜索.  
          2. 在调试过程中,由于遇到了智能体答非所问不知所云的问题,所以采用ReRank(重排序)的思想,引入ReRank模型Cross-Encoder Reranker,将检索后的5条信息进行筛选,得到最优的3个最终信息,经测评发现召回率提高了18%,大幅提高了用户检索内容的精确度.

    5. 强审核机制: 本智能体在决策过程中会经历"敏感词初审+输出审和+语义终审"三层审核机制,在保障用户信息安全的同时,维护良好的社区环境.

下面介绍云小圆，这是项目中的重点部分。首先介绍智能体的实现流程。云小圆不是一个简单调用接口的AI,而是汇集全平台工具于一身，能够帮助用户从头到尾完成所需工作闭环的Agent，系统将 LLM，也就是 Kimi，作为智能体的核心决策驱动，然后配备决策内循环、长短记忆、工具调用、意图控制展示和强审核机制，形成了一套深深嵌入项目业务的动态自适应图智能体架构。

决策内循环采用 ReAct 架构，基于 LangChain 和 LangGraph 制作了模型驱动、模型思考、模型观察的自我循环，使模型从一个输出者跃迁为一个决策者。

长短记忆方面，智能体的短期记忆采用“上下文裁剪 + 关键信息提取”的混合模式，长期记忆则采用 RAG 检索增强生成和 ChromaDB 向量库的模式。在这个过程中，关键对话信息和历史记录会通过 Embedding 模型转化为向量化坐标。当智能体调用搜索工具时，用户的问题也会被转化为同一向量空间中的查询向量，再在 ChromaDB 向量库中基于向量相似度检索 Top-K 语义邻近片段，从而实现意图搜索。

在调试过程中，由于遇到了智能体答非所问、不知所云的问题，所以采用 ReRank 重排序思想，引入 ReRank 模型 Cross-Encoder Reranker，将检索后的 5 条信息进行筛选，得到最优的 3 个最终信息。经测评发现，召回率提高了 18%，大幅提高了用户检索内容的精确度。

工具调用与意图控制展示方面，智能体不仅能回答用户的问题，还能调动系统内的合法工具，例如查询历史、解析文档与图片、获取个人偏好等。智能体还能通过 UI 控制协议，根据用户意图控制客户端的内容展示。例如，在解析文档时展示图谱小窗，在获取历史列表或搜索列表时展开列表。

强审核机制方面，本智能体在决策过程中会经历“敏感词初审 + 输出审核 + 语义终审”三层审核机制，在保障用户信息安全的同时，维护良好的社区环境。

接着进入公共数据大屏。大屏展示的不是静态演示数据，而是由 `/showcase/screen-data` 接口汇总真实业务数据，并定时刷新。这里主要用来说明平台已经具备运营观察能力，可以统计总用户、活跃用户、认证主体、总解析量、总评议量、总文件量、已审核文件、待审核文件、覆盖省份，以及解析和评议的小时峰值。

这些指标对应项目的三条核心链路。第一是用户和认证主体的参与链路，反映平台触达范围。第二是智能解析链路，反映政策文本进入 OCR、知识图谱、RAG 和智能体分析流程后的使用规模。第三是公众反馈链路，反映政策发布后的评价、纠错和办事留言。大屏还会计算文件公开率、主体渗透率、反馈密度和解析频度，用来判断政策从发布到理解、从阅读到反馈的闭环是否真正运转起来。

然后介绍登录注册的亮点。系统具备手机号登录和邮箱或用户名登录能力，属于 SaaS 系统形态。手机号登录和注册目前采用沙箱来模拟验证码，邮箱登录和注册加入了真实的 SMTP 邮件系统，注册流程也加入了图形验证码系统，以防止恶意访问。认证链路采用 JWT Access Token 做无状态会话鉴权，Token 使用 HS256 签名算法，并携带 sub、exp 等标准声明。密码侧采用 Passlib 的 PBKDF2-SHA256 哈希方案。验证码与图形验证码在服务端以 SHA-256 摘要存储，并配合 IP 维度、手机号或邮箱目标维度的频率限制、验证码 TTL、最大校验次数、邮箱验证状态与密码登录开关等机制形成多层防护。手机号一键登录在未来将真实获取用户设备有效信息，实现真实的一键登录逻辑。

进入系统后首先展示的是云小圆页面，这一部分稍后再讲，先切换到主页。

主页分为文件上传区和示例轮播区。示例区汲取轮播图 carousel 的灵感，通过 CSS Transform、Transition、overflow 裁切、层叠阴影、微角度 rotate 和 hover、active 状态反馈，让图片产生“超出边框但仍被容器约束”的动效层次，提高用户交互体验。文件上传区外形为卷轴式布局，用于上传桌面上的复杂政策文件。系统采用 PaddleOCR 或 LLM OCR 兜底的双层 OCR 保险机制，保障 OCR 过程的稳定性。

文件解析后，页面中展示可视化的发光知识图谱。图谱支持拖拽和节点查看，并在右侧显示父节点、子节点以及原文出处。右侧层级筛选可用于筛选特定层级的节点。右侧还会根据用户角色、职业画像、文档可执行性、时间风险和图谱复杂度，动态估计该政策对于用户的办理优先分数。核心算法可以展示为以下公式：

$$PriorityScore = 100 * (0.30 * ProfessionScore + 0.22 * RoleScore + 0.24 * ActionabilityScore + 0.14 * UrgencyScore + 0.10 * ComplexityScore) / TotalWeight$$

其中，ProfessionScore 来自职业关键词和扩展词命中，RoleScore 来自用户角色关键词命中，ActionabilityScore 来自“申请、申报、材料、入口”等可执行信号，UrgencyScore 来自截止时间和风险约束，ComplexityScore 则由节点数量、边数量、原文长度和负向约束边共同决定。

图谱支持全屏显示。右侧的转移标签可以让 LLM 根据用户需求个性化适配可视化解读，使解读更加大众化。播放能力可以语音播报文件核心信息。

平台还具备多模态解析能力。不仅可以解析 DOCX 等文件，还可以调用 OCR 系统解析 PDF 和图片。系统也配备了 URL 上传功能，可以调用爬虫和 Redis 缓存队列，在满足合法性的前提下获取公共政策文章内容并解析。

接下来介绍明暗切换和色系切换。系统通过全局 SCSS 变量系统性控制明暗两套色调，以及“深红 + 酒红 + 珊瑚”三套品牌色系，为用户提供良好的视觉体验。

随后进入民生大厅。民生大厅体现民生建设性反馈。页面正中间是民生反馈气泡式词云，当前采用关键词频次统计和前端力导向碰撞布局。系统先对评论内容按标点和空白切词，统计高频词 Top 28，再按词频映射字体大小、透明度和气泡半径。布局阶段使用 requestAnimationFrame 做中心引力、圆形碰撞检测、硬分离和边界回弹，保证气泡既有动态聚集感，又不会互相重叠。下方是民生反馈区，分为条形模式和砖块模式，并配备切页能力。

接下来介绍政策推荐页面。左侧展示企业和基层政策及通知，选中后在中间区域展示内容。下方可以点赞、收藏，也可以发布评论，评论经系统安全审核后可见。右侧展示中央文件和时事热点，并可跳转到相应页面。政策推荐基于用户画像关键词、职业扩展词、浏览记录、搜索记录和政策文本标签的混合推荐算法。系统先用角色和职业构建画像关键词集合，再对政策标题、分类、标签、正文和历史行为进行关键词命中与语义相似度打分。搜索侧采用“前缀匹配 + 子串/分词命中 + Embedding 语义相似度 + 意图加权”的混合排序，推荐侧再结合浏览热度与用户职业类别做重排，实现既能命中明确关键词，也能召回语义相关政策。

办事进度页面提供待办系统。用户可以新建待办，完成待办，删除待办，还可以根据自己的喜好切换是否显示已完成事项。该页面将用户对于政策的了解和学习，无缝衔接到实践环节。

接下来进入数据分析页面。这里重点说明平台如何把用户的每一次政策解析沉淀为可复盘的数据。普通用户看到个人解析数据，管理员可以对比个人与全体用户数据，因此它既服务个人效率反馈，也服务平台运营分析。

数据分析主要包括几类内容。第一是 RAG 命中趋势，用来统计检索增强过程中的查询次数、命中率、平均相似度和空检率。第二是节省时间估算，系统根据原文长度估算原始阅读时间，再与结构化解析后的阅读成本对比，得到总计节省时间和平均节省时间。第三是高频材料和通知类型分析，系统从解析结果中提取 required_materials 和 notice_type，并用 jieba 分词、停用词过滤和聚合统计找出高频材料、常见通知类型和用户处理重点。第四是通知难度评估，它会综合语言复杂度、办理复杂度和风险等级，帮助用户判断哪些政策更难理解、更需要优先处理。

下面介绍云小圆，这是项目中的重点部分。首先介绍智能体的实现流程。云小圆不是一个简单调用接口的AI,而是汇集全平台工具于一身，能够帮助用户从头到尾完成所需工作闭环的Agent，系统将 LLM，也就是 Kimi，作为智能体的核心决策驱动，然后配备决策内循环、长短记忆、工具调用、意图控制展示和强审核机制，形成了一套深深嵌入项目业务的动态自适应图智能体架构。

决策内循环采用 ReAct 架构，基于 LangChain 和 LangGraph 制作了模型驱动、模型思考、模型观察的自我循环，使模型从一个输出者跃迁为一个决策者。

长短记忆方面，智能体的短期记忆采用“上下文裁剪 + 关键信息提取”的混合模式，长期记忆则采用 RAG 检索增强生成和 ChromaDB 向量库的模式。在这个过程中，关键对话信息和历史记录会通过 Embedding 模型转化为向量化坐标。当智能体调用搜索工具时，用户的问题也会被转化为同一向量空间中的查询向量，再在 ChromaDB 向量库中基于向量相似度检索 Top-K 语义邻近片段，从而实现意图搜索。

在调试过程中，由于遇到了智能体答非所问、不知所云的问题，所以采用 ReRank 重排序思想，引入 ReRank 模型 Cross-Encoder Reranker，将检索后的 5 条信息进行筛选，得到最优的 3 个最终信息。经测评发现，召回率提高了 18%，大幅提高了用户检索内容的精确度。

工具调用与意图控制展示方面，智能体不仅能回答用户的问题，还能调动系统内的合法工具，例如查询历史、解析文档与图片、获取个人偏好等。智能体还能通过 UI 控制协议，根据用户意图控制客户端的内容展示。例如，在解析文档时展示图谱小窗，在获取历史列表或搜索列表时展开列表。

强审核机制方面，本智能体在决策过程中会经历“敏感词初审 + 输出审核 + 语义终审”三层审核机制，在保障用户信息安全的同时，维护良好的社区环境。

接下来介绍搜索框。搜索框采用聚合搜索，结合关键字搜索、Trie 前缀搜索和 RAG 意图检索，从一切历史记录中搜索。它不仅可以搜索浏览记录，也可以搜索解析记录，甚至可以搜索对话记录，达到搜索一切的效果。输入“城市”时，会出现语义搜索结果，包括“公租房”“城乡”，以及前缀搜索内容中包含“城市”的内容。

随后进入管理员账号。

政策发布中心支持管理员和认证主体上传自身的企业政策。用户可以手动填写，也可以上传文件。文件上传后，系统启动 OCR 识别文件文字，并自动填充到文件标题和正文。提交审核后，经管理员审核即可发布到政策大厅。

管理员页面以可视化形式展示全站的各类信息，包括注册和活跃用户数量、节省时间趋势、用户角色分布、热词 Top20 等。管理员还可以在这里管理普通用户，例如冻结用户、设为管理员等。设置后，系统将通过 SMTP 邮件系统向该用户邮箱内发送升级邮件。

个人中心是个人信息的展示页面，陈列个人的各类数字内容，例如个人卡片、快捷入口、最近解析、最近收藏等。用户可以切换头像，也可以上传自己喜爱的头像。用户还可以查看个人画像，了解自己的政策偏好。

最后介绍系统设置。用户可以在这里更改默认编写受众，切换配色和主题，绑定手机号和邮箱，设置登录密码等。

# 录屏操作脚本

1. 打开首页，依次展示 10 张动效页面。
2. 切换到政策广场，展示功能中心、资讯列表、热点文件和全景政策广场。
3. 在功能中心选择一个砖块入口，展示页面跳转效果。
4. 切换资讯列表的列表模式和卡片模式。
5. 通过首页底部按钮切换到数据大屏，展示公共数据大屏的各项指数。
6. 进入登录页，展示手机号登录、邮箱或用户名登录、注册流程和图形验证码。
7. 使用用户级账号登录并进入系统。
8. 登录后短暂停留云小圆页面，然后切换到主页。
9. 展示示例轮播区的动效。
10. 在文件上传区上传一份复杂政策文件。
11. 等待文件解析完成，展示发光知识图谱。
12. 拖拽图谱节点，并查看右侧父节点、子节点和原文出处。
13. 使用右侧层级筛选，筛选特定层级节点。
14. 展示右侧办理优先分数和核心公式。
15. 进入图谱全屏模式，再退出全屏。
16. 切换右侧转移标签，展示个性化可视化解读。
17. 播放语音播报，展示文件核心信息播报效果。
18. 展示 DOCX、PDF、图片和 URL 上传入口。
19. 切换明暗模式。
20. 切换品牌色系，展示不同配色效果。
21. 进入民生大厅，展示气泡式词云。
22. 切换民生反馈区的条形模式和砖块模式。
23. 展示民生反馈区切页效果。
24. 进入政策推荐页面。
25. 选择左侧一条企业或基层政策，在中间区域展示内容。
26. 展示点赞、收藏和评论入口。
27. 选择右侧中央文件或时事热点，展示页面跳转效果。
28. 进入办事进度页面。
29. 新建一个待办事项。
30. 完成并删除待办事项。
31. 切换是否显示已完成事项。
32. 进入数据分析页面，简要展示主要数据模块。
33. 进入云小圆页面。
34. 提问“你是谁，拥有哪些技能？”
35. 提问“我浏览过哪些政策？”
36. 提问“你觉得我的个人画像是怎样？”
37. 提问“帮我在历史记录中搜索‘企业’。”
38. 提问“帮我解析这个文档。”
39. 打开搜索框，逐字输入“城市”。
40. 展示“公租房”“城乡”以及包含“城市”的前缀搜索结果。
41. 切换到管理员账号。
42. 进入政策发布中心。
43. 手动填写一条政策内容。
44. 上传一份政策文件，展示 OCR 自动识别并填充标题和正文。
45. 提交审核，并展示审核发布流程。
46. 进入管理员页面，展示全站统计信息。
47. 展示普通用户管理功能，例如冻结用户和设为管理员。
48. 展示 SMTP 邮件升级通知效果。
49. 进入个人中心，展示个人卡片、快捷入口、最近解析和最近收藏。
50. 切换或上传头像。
51. 查看个人画像。
52. 进入系统设置。
53. 展示默认编写受众、配色主题、手机号绑定、邮箱绑定和登录密码设置。
