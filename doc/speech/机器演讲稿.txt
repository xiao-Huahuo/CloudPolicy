尊敬的评委老师大家好,接下来要介绍的我们团队开发的一款民生政策与智能服务一体化平台------云枢观策平台.
你是否曾经心力交瘁地逐字阅读红头文件,却难以Get到政策的核心与宗旨?
你是否曾经发布文章,却难以得到用户切真而有效的反馈?

云枢观策致力于以人工智能赋能政策信息流转链路，构建“共享、解读、服务、协同”一体化平台，让企业与基层政策更加可达、可懂、可办、可行。

平台业务从企业政策发布,到基层服务的落地形成完整闭环。最终由“云小圆”智能体集成。

平台由客户端与云服务端组成,并引入了智能体协同架构.客户端采用了响应式布局与多端适配,保证了不同群体客户的最佳体验.

下面进入项目演示视频部分。

首先来到展示首页。它把政策发布、公众理解、智能解析、民生反馈和数据治理放在同一条链路里，让用户先理解项目要解决的问题，再进入实际业务系统。

首页中依次概括了项目的核心能力，包括智能检索、数据大屏、Agent 对话、政策广场、个人中心、智能解析、民生大厅、认证主体服务和可视化政策图谱。这里的重点不是展示页面本身，而是说明平台已经形成从“发现政策”到“理解政策”，再到“办理事项、反馈沉淀、数据复盘”的完整闭环。同时，首页中的政策数量、注册用户和参与评议等指标来自真实接口，能够反映系统已有的数据沉淀。

接下来进入政策广场。政策广场包含功能中心。功能中心采用砖块墙布局，可跳转到相应页面。资讯列表和热点文件区域用于展示最新的中央和地方政策。数据来源采用“索引跳转 + 本地兜底数据”的双层策略。中央文件主要接入中国政府网 RSS 链接，时事热点接入人民日报、新华社、央视新闻等公共 RSS 源的链接，并通过 Redis 缓存和内存缓存降低重复请求。当外部网络不可用时，系统会自动回退到本地演示数据，保障演示稳定性。该区域具备列表和卡片两种展示形式。第三是全景政策广场，用于展示全站认证主体，包括企业和地方惠民机构推送的政策。这里包括企业、社区、街道及地方惠民机构推送并审核通过的政策内容。该模块作为平台的公共政策入口，集中承载政策公开、分类浏览、智能解读、用户评价与反馈沉淀等功能，帮助用户快速发现与自身相关的政策信息，也帮助发布主体了解政策触达效果和群众关注重点，形成从政策发布到服务优化的闭环。

接着进入公共数据大屏。大屏展示的不是静态演示数据，而是由 `/showcase/screen-data` 接口汇总真实业务数据，并定时刷新。这里主要用来说明平台已经具备运营观察能力，可以统计总用户、活跃用户、认证主体、总解析量、总评议量、总文件量、已审核文件、待审核文件、覆盖省份，以及解析和评议的小时峰值。

这些指标对应项目的三条核心链路。第一是用户和认证主体的参与链路，反映平台触达范围。第二是智能解析链路，反映政策文本进入 OCR、知识图谱、RAG 和智能体分析流程后的使用规模。第三是公众反馈链路，反映政策发布后的评价、纠错和办事留言。大屏还会计算文件公开率、主体渗透率、反馈密度和解析频度，用来判断政策从发布到理解、从阅读到反馈的闭环是否真正运转起来。

然后介绍登录注册的亮点。系统具备手机号登录和邮箱或用户名登录能力，属于 SaaS 系统形态。手机号登录和注册目前采用沙箱来模拟验证码，邮箱登录和注册加入了真实的 SMTP 邮件系统，注册流程也加入了图形验证码系统，以防止恶意访问。认证链路采用 JWT Access Token 做无状态会话鉴权，Token 使用 HS256 签名算法，并携带 sub、exp 等标准声明。密码侧采用 Passlib 的 PBKDF2-SHA256 哈希方案。验证码与图形验证码在服务端以 SHA-256 摘要存储，并配合 IP 维度、手机号或邮箱目标维度的频率限制、验证码 TTL、最大校验次数、邮箱验证状态与密码登录开关等机制形成多层防护。手机号一键登录在未来将真实获取用户设备有效信息，实现真实的一键登录逻辑。

进入系统后首先展示的是云小圆页面，这一部分稍后再讲，先切换到主页。

主页分为文件上传区和示例轮播区。示例区汲取轮播图 carousel 的灵感，通过 CSS Transform、Transition、overflow 裁切、层叠阴影、微角度 rotate 和 hover、active 状态反馈，让图片产生“超出边框但仍被容器约束”的动效层次，提高用户交互体验。文件上传区外形为卷轴式布局，用于上传桌面上的复杂政策文件。系统采用 PaddleOCR 或 LLM OCR 兜底的双层 OCR 保险机制，保障 OCR 过程的稳定性。

文件解析后，页面中展示可视化的发光知识图谱。图谱支持拖拽和节点查看，并在右侧显示父节点、子节点以及原文出处。右侧层级筛选可用于筛选特定层级的节点。右侧还会根据用户角色、职业画像、文档可执行性、时间风险和图谱复杂度，动态估计该政策对于用户的办理优先分数。核心算法可以展示为以下公式：

$$Priority = 100 * (0.30 * Profession + 0.22 * Role + 0.24 * Actionability + 0.14 * Urgency + 0.10 * Complexity) / Total$$  

其中 Prof 来自职业关键词命中,Role 来自用户关键词命中,Action 来自可执行信号,Urgency 来自截止时间,Complex 则由节点数量、边数量、原文长度和负向约束边共同决定.  

图谱支持全屏显示。右侧的转移标签可以让 LLM 根据用户需求个性化适配可视化解读，使解读更加大众化。播放能力可以语音播报文件核心信息。
图谱还支持 Trie 前缀搜索，可以根据用户输入的前几个字快速匹配相关节点，帮助用户在大规模政策图谱中迅速定位目标内容。

平台具备多模态解析能力。不仅可以解析 DOCX 等文件，还可以调用 OCR 系统解析 PDF 和图片。系统也配备了 URL 上传功能，可以调用爬虫和 Redis 缓存队列，在满足合法性的前提下获取公共政策文章内容并解析。

平台配备了全局性的明暗切换和三色系切换。系统通过全局 SCSS 变量系统性控制明暗两套色调，以及“深红 + 酒红 + 珊瑚”三套品牌色系，为用户提供良好的视觉体验。

随后进入民生大厅。民生大厅体现民生建设性反馈。页面正中间是民生反馈气泡式词云，当前采用关键词频次统计和前端力导向碰撞布局。系统先对评论内容按标点和空白切词，统计高频词 Top 28，再按词频映射字体大小、透明度和气泡半径。布局阶段使用 requestAnimationFrame 做中心引力、圆形碰撞检测、硬分离和边界回弹，保证气泡既有动态聚集感，又不会互相重叠。下方是民生反馈区，分为条形模式和砖块模式，并配备切页能力。

民生大厅用于汇聚群众对政策落地后的评价、纠错和办事留言，把分散的反馈集中沉淀下来，帮助用户看到政策执行中的真实体验，也帮助发布主体及时发现问题、调整服务，形成从政策发布到反馈治理的闭环。

接下来介绍政策推荐页面。左侧展示企业和基层政策及通知，选中后在中间区域展示内容。下方可以点赞、收藏，也可以发布评论，评论经系统安全审核后可见。右侧展示中央文件和时事热点，并可跳转到相应页面。
政策推荐采用“职业类别优先 + 浏览热度排序”的轻量推荐算法。优先推荐该分类下浏览量更高的已审核政策，再补充其他高浏览量政策，从而兼顾个性相关性和公共热度。
系统先用角色和职业构建画像关键词集合，再对政策标题、分类、标签、正文和历史行为进行关键词命中与语义相似度打分。搜索侧采用“前缀匹配 + 子串/分词命中 + Embedding 语义相似度 + 意图加权”的混合排序，推荐侧再结合浏览热度与用户职业类别做重排，实现既能命中明确关键词，也能召回语义相关政策。

办事进度页面提供待办系统。用户可以新建待办，完成待办，删除待办，还可以根据自己的喜好切换是否显示已完成事项。办事中心将政策理论转向实践,把后续动作拆解成可追踪的待办事项,让政策不再只是停留在文件、摘要和图谱中的知识，而是变成一个个可以执行、可以完成、可以回看的现实步骤。

接下来进入数据分析页面。这里重点说明平台如何把用户的每一次政策解析沉淀为可复盘的数据。普通用户看到个人解析数据，管理员可以对比个人与全体用户数据，因此它既服务个人效率反馈，也服务平台运营分析。

数据分析主要包括四类内容：
第一，RAG 命中趋势，统计查询次数、命中率、平均相似度和空检率。
第二，节省时间估算，对比原文阅读时间与结构化解析后的阅读成本，计算总节省时间和平均节省时间。
第三，高频材料与通知类型分析，提取 required_materials 和 notice_type，通过 jieba 分词、停用词过滤和聚合统计，分析高频材料、常见通知类型和用户处理重点。
第四，通知难度评估，综合语言复杂度、办理复杂度和风险等级，判断政策理解难度和处理优先级。

下面介绍云小圆智能体，这是项目中的重点部分。首先介绍智能体的实现流程。云小圆不是一个简单调用接口的AI,而是汇集全平台工具于一身，能够帮助用户从头到尾完成所需工作闭环的Agent，系统将 LLM，也就是 Kimi，作为智能体的核心决策驱动，然后配备决策内循环、长短记忆、工具调用、意图控制展示和强审核机制，形成了一套深深嵌入项目业务的动态自适应图智能体架构。

决策内循环采用 ReAct 架构，基于 LangChain 和 LangGraph 制作了模型驱动、模型思考、模型观察的自我循环，使模型从一个输出者跃迁为一个决策者。

长短记忆方面，智能体的短期记忆采用“上下文裁剪 + 关键信息提取”的混合模式，长期记忆则采用 RAG 检索增强生成和 ChromaDB 向量库的模式。在这个过程中，关键政策对话信息和历史记录会通过 Embedding 模型转化为向量化坐标。当智能体调用搜索工具时，用户的问题也会被转化为同一向量空间中的查询向量，再在 ChromaDB 向量库中基于向量相似度检索 Top-K 语义邻近片段，从而实现意图搜索。

在调试过程中，由于遇到了智能体答非所问、不知所云的问题，所以采用 ReRank 重排序思想，引入 ReRank 模型 Cross-Encoder Reranker，将检索后的 5 条信息进行筛选，得到最优的 3 个最终信息。经测评发现，召回率提高了 18%，大幅提高了用户检索内容的精确度。

工具调用与意图控制展示方面，智能体不仅能回答用户的问题，还能调动系统内的合法工具，例如查询历史、解析文档与图片、获取个人偏好等。智能体还能通过 UI 控制协议，根据用户意图控制客户端的内容展示。例如，在解析文档时展示图谱小窗，在获取历史列表或搜索列表时展开列表。

强审核机制方面，本智能体在决策过程中会经历“敏感词初审 + 输出审核 + 语义终审”三层审核机制，在保障用户信息安全的同时，维护良好的社区环境。

接下来介绍搜索框。搜索框采用聚合搜索，结合关键字搜索、Trie 前缀搜索和 RAG 意图检索，从一切历史记录中搜索。它不仅可以搜索浏览记录，也可以搜索解析记录，甚至可以搜索对话记录，达到搜索一切的效果。输入“城市”时，会出现语义搜索结果，包括“公租房”“城乡”，以及前缀搜索内容中包含“城市”的内容。

随后进入管理员账号。

政策发布中心支持管理员和认证主体上传自身的企业政策。认证主体或管理员可以手动填写政策内容。未来将开通上传文件的功能文件上传后，系统启动 OCR 识别文件文字，并自动填充到文件标题和正文。提交审核后，经管理员审核即可发布到政策大厅。

管理员页面以可视化形式展示全站的各类信息，包括注册和活跃用户数量、节省时间趋势、用户角色分布、热词 Top20 等。管理员还可以在这里管理普通用户，例如冻结用户、设为管理员等。设置后，系统将通过 SMTP 邮件系统向该用户邮箱内发送升级邮件。

个人中心是个人信息的展示页面，陈列个人的各类数字内容，例如个人卡片、快捷入口、最近解析、最近收藏等。用户可以切换头像，也可以上传自己喜爱的头像。用户还可以查看个人画像，了解自己的政策偏好。

最后介绍系统设置。用户可以在这里更改默认编写受众，切换配色和主题，绑定手机号和邮箱，设置登录密码等。
