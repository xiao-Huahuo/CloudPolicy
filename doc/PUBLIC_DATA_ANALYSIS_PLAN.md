# 公共数据分析大屏重构实施方案

## 目标

彻底删除当前 `ShowcaseScreen.vue` 中偏占位、偏普通后台的 widget 布局，重建一个以“繁华、科技、展示、复杂分析感”为核心的大屏页面。  
风格上与首页首屏右侧的“政策脉冲”模块统一，采用：

- 深酒红紫 + 深蓝黑作为主背景
- 珊瑚暖橙作为品牌主高光
- 冷青蓝作为数据辅助高光
- 少量荧光薄荷绿作为状态色

页面定位不是“仪表盘”，而是“可以直接拿去展示、汇报、路演、放到大屏幕墙上”的公共数据分析舞台。

---

## 视觉方向

### 1. 总体风格

- 禁止使用普通圆角白底简约卡片
- 全部模块使用切角、机械托底、流光边框、镜面暗色材质
- 标题必须有“承托结构”，看起来像被一块机械底座或能量轨道托起
- 布局以“中间地图主舞台 + 四周复杂分析模块”展开

### 2. 配色体系

- 背景主渐变：`#140c12` → `#512334` → `#15233f` → `#09131f`
- 珊瑚高光：`#ff8f7a`、`#ffb76f`
- 冷青高光：`#58cbff`、`#5fd1ff`
- 状态绿：`#80fab0`
- 预警黄：`#ffd166`
- 危险红：`#ff6b6b`

### 3. 动效原则

- 所有动效都要“平滑、持续、低噪声”，不要廉价抖动
- 流光边框、扫描线、数据脉冲、标题下托座、地图外圈轨道均持续轻动
- 交互反馈必须有，但不能有首页那种已经被删掉的回弹感

## 组件拆分方案

页面组件：

- `web/src/views/showcase/ShowcaseScreen.vue`

子组件目录：

- `web/src/components/showcase_screen/`

计划组件：

1. `ScreenTopBar.vue`
   - 大屏总标题
   - 托底标题结构
   - 时间、数据源、状态灯

2. `LuminousFrame.vue`
   - 通用流光边框容器
   - 切角造型
   - 动态边缘流光
   - 所有分析模块复用

3. `FrameTitle.vue`
   - 模块标题
   - 带托底承台、扫描线、左右机械装饰

4. `DataTower.vue`
   - 基于 `web/super-comonents/数据显示塔.html` 重构
   - 接受颜色、数值、标签、百分比
   - 一屏展示 3 个不同颜色的数据塔

5. `DataTowerGroup.vue`
   - 管理 3 个数据显示塔布局
   - 用于大屏左上主指标

6. `ChinaMapDeck.vue`
   - 核心地图组件
   - 内部仍使用 ECharts 地图实现省份缩放/拖拽
   - 外层通过 CSS 3D + 指针交互实现三维移动
   - 通过伪层/底层复制制造厚度
   - 加入流光扫掠、环形轨道、发光描边

7. `TrendMatrix.vue`
   - 趋势折线或面积图
   - 用于解析量、活跃度、反馈脉冲

8. `DistributionPanel.vue`
   - 用于角色分布、评议类型分布
   - 可以内部切换饼图 / 横向能量条

9. `SignalFeed.vue`
   - 右侧实时动态流
   - 自动追加滚动
   - 带发光状态点和时间戳

10. `SummaryRibbon.vue`
    - 底部总结带
    - 横向跑动数据摘要

## 地图技术方案

### 目标能力

- 保留管理员页面那种可缩放、可拖拽中国地图
- 增加三维移动感
- 增加厚度感
- 增加流光扫描与外圈轨道

### 实现策略

采用“ECharts 2D 地图 + 外层 3D 舞台包装”的方案，而不是直接上真正三维 GIS：

1. 地图主体仍然使用 ECharts `map`
   - 这样可以稳定保留省份 hover、tooltip、roam 缩放和平移

2. 地图舞台外层使用 `perspective` 和 `transform-style: preserve-3d`
   - 鼠标移动时平滑改变 `rotateX / rotateY`
   - 形成可三维观察的感觉

3. 厚度通过 2 到 3 层伪底板实现
   - 主地图层
   - 下沉阴影层
   - 发光描边底层

4. 流光效果通过以下叠加实现
   - 外围轨道环绕
   - 地图表面扫描束
   - 高亮省份脉冲散点
   - 边缘 glow 阴影

### 风险说明

- 当前项目没有引入 `echarts-gl`，不走真正 3D 地图，避免依赖膨胀和高开发风险
- 该方案能满足“看上去可三维移动、有厚度、有科技展示感”的核心目标

## 数据方案

当前 `ShowcaseScreen.vue` 依赖管理员接口，不适合公开展示页。  
因此新增公开接口：

- `GET /showcase/screen-data`

返回内容建议：

- `summary`
  - 总用户数
  - 认证主体数
  - 总解析量
  - 总评议数
  - 总文件数
  - 已审核文件数
  - 待审核文件数
  - 活跃用户数

- `role_dist`
  - 普通用户 / 认证主体 / 管理员

- `geo_dist`
  - 省份分布

- `opinion_type_dist`
  - review / correction / message

- `rating_dist`
  - 1 到 5 星

- `hourly_trend`
  - 最近 24 小时解析量

- `hourly_opinion_trend`
  - 最近 24 小时评议量

- `feed`
  - 可展示的实时摘要文本

数据不足时允许后端根据现有真实数据做合理聚合，不允许前端再用纯随机数冒充核心指标。
